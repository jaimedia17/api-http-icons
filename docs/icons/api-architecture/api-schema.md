# API Schema Icon  
Structured Data Model and Payload Definition Layer

## Overview  
This icon represents the API Schema layer within the API boundary, using the same abstract, protocol‑neutral visual language applied across the API/HTTP icon system.  
The API Schema defines the structural shape of request and response payloads, describing fields, types, and constraints that make up the data exchanged with the API.

The icon expresses this by using a dense set of internal horizontal lines inside the schema block, showing that the API exposes a detailed, declarative data model.

## Geometric Meaning  
**Circular boundary** — represents the API surface where structured data enters and exits the system.  
**Internal schema block** — the rounded rectangle inside the circle symbolizes the data model, containing the detailed structure of the payload.  
**High‑density internal lines** — represent fields, types, and constraints that define the shape of the data.  
**Inbound lane** — the single bar entering from the left represents a request carrying structured data.  
**Upstream fan‑out bars** — the horizontal bars on the right represent the structured response or mapped data leaving the API.  
**Dense segmentation** — communicates that the schema contains many fields and detailed structure, distinguishing it from the simpler contract icon.

## Schema Semantics  
The internal density is the core semantic signal of the API Schema concept.  
It represents the detailed structure of the payload: fields, data types, constraints, and shape.

Unlike the contract icon, which expresses high‑level agreement, the schema icon communicates low‑level structure and data modeling.  
This abstraction aligns with real API behavior such as serialization, validation, mapping, and schema‑driven integration.

## Narrative Fit  
The icon mirrors the real technical sequence of schema‑driven API communication:  
A client sends a structured payload, the API interprets it according to the schema, and the system returns a structured response.

No routing, transformation, or gateway logic is implied; the icon represents a functioning API surface with a defined data model.  
This abstraction aligns with how APIs define payload structure, enforce data types, and ensure consistent serialization across distributed systems.
