# API Key Icon  
Machine-Issued Credential for Authentication and Access Control

## Overview  
This icon represents an API Key or machine-issued credential used to authenticate and authorize access to system resources.  
It follows the same abstract, protocol-neutral visual language applied across the API/HTTP icon system.

An API Key is a token that identifies a client, defines its permissions, and grants controlled access to backend services.  
The icon expresses this by using a compact, symbolic credential form that communicates identity, scope, and access level without relying on literal key imagery.

## Geometric Meaning  
**Rounded square boundary** — represents the credential container, a secure, self-contained token issued by the system.  
**Dual-circle markers** — symbolize identity and scope: the key’s unique identifier and its permission set.  
**Vertical access bars** — represent the defined access rules, such as read/write permissions or service-level entitlements.  
**Minimalist token form** — communicates that the credential is machine-generated, structured, and used for automated authentication.

## Credential Semantics  
The internal arrangement of circles and bars is the core semantic signal of the API Key concept.

It represents identity, scope, permissions, and access control.  
Unlike authorization tokens or OAuth flows, this icon communicates a static, pre-issued credential used for direct API access.

This abstraction aligns with real API behavior such as header-based authentication, service-to-service credentials, and backend integration keys.

## Narrative Fit  
The icon mirrors the real technical sequence of credential-based access:

A client presents an API Key →  
The system validates the key →  
Permissions and scope are checked →  
Access is granted to the appropriate backend resources.

No session management, token refresh, or multi-step negotiation is implied.  
The icon represents a simple, deterministic credential used in many API architectures.
