# 408 — Request Timeout Icon  
Client Request Exceeded Allowed Time Window

## Overview  
This icon represents the 408 Request Timeout condition using the same abstract, protocol‑agnostic visual language used across the API icon system.  
A 408 error occurs when the server begins waiting for the remainder of a client request, but the client does not finish sending data within the allowed time window.

The failure is not structural, not credential‑related, and not resource‑related.  
Instead, it is a progress interruption.  
The icon expresses this by placing the interruption between the credential boundary and resource layer, showing that the request enters the system but stalls before reaching the endpoint.

## Geometric Meaning  
The left circle represents a valid, well‑formed request.  
The solid left bar indicates that the request successfully reaches the server boundary and passes initial validation.

The two internal blocks between the bars represent the stalled progression.  
They show that the server is waiting for additional data that never arrives.

The right bar confirms that the endpoint exists and is reachable, but the request never progresses far enough to interact with it.

This geometry mirrors the real technical sequence:  
the request is valid, the server is ready, but the client does not complete transmission.

## Stalled Progress Indicator  
The two blocks between the bars are the visual cue for timeout.  
They do not represent failure at the origin or failure at the resource.  
Instead, they represent incomplete transit — a request that begins correctly but never finishes.

Their placement between the bars confirms that the failure occurs during the request lifecycle, not before or after.

## Narrative Fit  
The icon aligns with the true meaning of 408:  
the server waits, but the client does not complete the request.  
The system is ready, the endpoint exists, and the request is structurally valid — but the process halts mid‑flow.

This abstraction fits the semantics of 408 without relying on clocks, timers, or literal metaphors, keeping the design consistent with the rest of the API icon family.
