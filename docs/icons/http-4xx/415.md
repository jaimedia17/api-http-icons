# 415 — Unsupported Media Type Icon  
Payload Format Not Supported by Server

## Overview  
This icon represents the 415 Unsupported Media Type condition using the same abstract geometric language used throughout the HTTP icon system.  
A 415 error occurs when the server receives a request body encoded in a format it does not support.  
The request is structurally valid, but the payload’s media type is incompatible with the server’s expectations.

The icon expresses this by giving the payload block a mismatched geometric shape, indicating that the format itself is invalid before evaluation.

## Geometric Meaning  
The circle on the left represents the incoming request.  
Attached to it is the payload block, which features a deliberate shape mismatch on its left side.  
This cut or slant indicates the payload format is malformed or incompatible.

The payload is intentionally separated from the server’s evaluation sign, showing that the server rejects the request before attempting to process or interpret the body.

The resource layer on the right remains untouched, as the server cannot proceed without a supported media type.

## Unsupported Format Indicator  
The geometric mismatch on the payload block is the core semantic signal of the 415 condition.  
Unlike 413, where the payload is too large, the 415 payload is the wrong shape.  
This communicates that the issue is not size but format.

The mismatch is placed on the payload itself to show that the failure originates in the request body’s encoding, not in the server’s layers or the resource.

## Narrative Fit  
The icon mirrors the real technical sequence of a 415 error.  
The request exists and reaches the server, but the server cannot interpret the payload because its media type is unsupported.

The payload is rejected before evaluation, the resource layer remains irrelevant, and the server returns an Unsupported Media Type response.

This abstraction aligns with how APIs enforce content‑type validation in real workflows.
