# 422 — Unprocessable Entity Icon  
Semantically Invalid Payload Rejected by Server

## Overview  
This icon represents a 422 Unprocessable Entity condition using abstract, geometric signal geometry.  
A 422 error occurs when the request is structurally valid and successfully reaches the server but cannot process it because the payload is semantically invalid, incomplete, or fails validation.  
The request understood the syntax but cannot adhere to the semantic rules.

It also encapsulates the inability to follow the request data rather than the resource data, because the problem originates from the client data itself, making it not functional at the resource level.

## Geometric Meaning  
The circle in the icon represents the incoming request.  
It has 422 essence: the request is well‑formed, delivered successfully, and structurally valid.

The slash in it is the substance of the data requirement that cannot be fulfilled when the request violates the expected semantics.

The horizontal bar represents the server’s inability to process the request due to semantic failure.  
The bar cuts through the circle, showing that although the request reaches the server, the data cannot be processed.

The geometric composition thus visually conveys the concept of an unprocessable entity.  
The clean lines and minimalistic design emphasize the clarity of the error condition.  
The absence of color gradients or embellishments reinforces the concept’s technical nature.  
The viewer understands the agent, object, and process through the icon’s form, which is derived from the standard design language of API error visuals.

## Invalid Payload Indicator  
The slash symbol in place of the request data represents the semantic issue within the payload.  
This includes invalid fields, malformed values, missing required data, or semantic errors such as data type mismatches.

The request reaches the server but cannot be executed meaningfully.  
The placement of the failure indicator communicates that the breakdown occurs after syntax validation but before any output or change at resource level.

## Narrative Fit  
The icon’s narrative is a graphical interpretation of a 422 error.  
The request was sent and structurally valid.  
The credentials are accepted, and the request passes the authentication boundary.

However, the payload cannot be processed due to semantic issues.  
The circle and bar combination visually narrate this breakdown.  
The circle represents the request’s completeness, while the bar represents the unprocessable nature of the payload.

The design fits within the family of HTTP status code visuals, such as 404 (Not Found), 500 (Internal Server Error), and 403 (Forbidden).  
It maintains consistency with the API HTTP workflow icon set, ensuring clarity and recognizability.
