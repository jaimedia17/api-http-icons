# 410 — Gone Icon  
Resource Permanently Removed

## Overview  
This icon represents the **410 Gone** condition using a clean, abstract geometric structure.  
A 410 error indicates that the resource previously existed but has been permanently removed.  
Unlike 404, which expresses uncertainty or absence, 410 communicates **intentional deletion**.

The icon expresses this by placing the failure inside the resource layer, showing that the request is valid, the endpoint exists, and the system understands what is being asked for — but the resource itself is no longer present.

## Geometric Meaning  
The left circle represents a valid request.  
The solid left bar indicates that the request passes authentication and validation.

The right bar represents the resource layer — the endpoint is known and reachable.

The internal missing segment inside the right bar is the defining semantic element.  
It represents a resource that used to exist but has been permanently removed.

This internal void is not a credential failure, not a malformed request, and not a missing endpoint.  
It is specifically a **removed resource**, which is the core meaning of 410.

## Permanent Removal Indicator  
The missing internal segment is the visual cue for “gone.”

It is placed inside the right bar to show that the failure occurs at the resource layer, not at the request origin or during processing.

Its shape differentiates it from 404, which uses a circular void to represent an unknown or unresolvable resource.

410’s rectangular internal absence communicates a deliberate, permanent removal.

## Narrative Fit  
The icon informs the real technical sequence of a 410 error.  
The request is valid, the credentials are valid, and the endpoint exists.  
The system knows exactly what resource is being requested — but that resource has been intentionally removed.

This abstraction aligns with the semantics of 410 without relying on literal metaphors, maintaining consistency with the rest of the API icon system.
