# 409 — Conflict Icon  
Resource State Prevents Requested Operation

## Overview  
This icon represents the 409 Conflict condition using the same clean, abstract visual language applied across the entire error family.  
A 409 error occurs when the request is valid and reaches the resource layer, but the operation cannot proceed because the resource’s current state conflicts with the requested action.  
The server recognizes the request and the credentials, but the internal state of the target prevents the operation from completing.

The icon expresses this by placing the failure inside the resource layer, showing that the conflict originates from the resource’s internal conditions rather than from authentication or request structure.

## Geometric Meaning  
The circle on the left represents the incoming request.  
In a 409 scenario, the request is structurally valid and passes the authentication boundary without issue.

The middle bar represents the credential layer, which remains unobstructed because the server accepts the identity and structure of the request.

The right bar represents the target resource.  
This is where the failure occurs.  
The request reaches the resource, but the operation cannot proceed due to a conflict with the resource’s current state.

Inside the right bar, a single vertical obstruction block is placed centrally.  
This obstruction spans the full height of the resource bar and remains inset from both edges.  
This placement indicates that the resource exists and is reachable, but its internal state prevents the requested operation from being applied.

## Conflict Indicator  
The obstruction block inside the resource bar represents the internal conflict.  
It shows that the resource is present and accessible, but the server cannot perform the requested action because the resource’s current state is incompatible with the operation.

Unlike 403, where the boundary is blocked, or 410, where part of the resource is missing, the 409 obstruction is entirely internal.  
This symbol communicates that the failure is not about identity, permissions, or availability, but about the resource’s internal conditions at the moment of the request.

## Narrative Fit  
The icon mirrors the real technical sequence of a 409 error.  
The request is valid, authenticated, and reaches the resource layer.  
The server evaluates the resource and determines that the operation cannot be completed due to a state conflict — such as version mismatches, concurrent modifications, or incompatible updates.

The resource remains intact, the boundary remains open, and the failure is tied directly to the internal state of the resource.

This abstraction aligns with the true meaning of 409 in API and HTTP workflows:  
the request cannot be fulfilled because the resource’s current state conflicts with the requested operation.
