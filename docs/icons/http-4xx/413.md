# 413 — Payload Too Large Icon  
Oversized Request Body Blocked by Server Gate

## Overview  
This icon represents the 413 Payload Too Large condition using the same abstract, protocol‑neutral visual language used across the entire HTTP icon system.  
A 413 error occurs when the client sends a request body that exceeds the server’s configured size limits.  
The request is structurally valid, but the payload is too large to be accepted or processed.

The icon expresses this by showing the payload block physically wider than the server’s evaluation gate, making it impossible for the request to proceed.

## Geometric Meaning  
The circle on the left represents the incoming request.  
Attached to it is the payload block, which is intentionally oversized.  
This exaggerated width is the core semantic signal of the 413 condition.

The left bar represents the server’s evaluation gate.  
Because the payload is wider than the gate, the request cannot pass through.

The resource layer on the right remains untouched, indicating that the server never evaluates or interacts with the target resource due to the excessive payload size.

## Oversized Payload Indicator  
The payload block is deliberately drawn wider than the gate to communicate the size violation.  
This is not a symbolic gesture but a geometric constraint: the payload physically cannot fit through the server boundary.

This abstraction mirrors the real technical behavior of a 413 error, where the server rejects the request before any further processing because the body exceeds allowed limits.

## Narrative Fit  
The icon mirrors the actual sequence of a 413 error.  
The request exists, the payload is present, and the server receives it.  
However, the server immediately rejects the request because the payload is too large to process.

The resource layer remains irrelevant, and the server returns a Payload Too Large response.  
This abstraction aligns with how APIs and HTTP servers enforce body size limits in real workflows.
