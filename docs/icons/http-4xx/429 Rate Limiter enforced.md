# 429 — Rate Limiter (Enforced) Icon  
System‑Level Throttling and Request Flow Control

## Overview  
This icon represents the enforcement side of HTTP 429 Rate Limiting.  
The geometry expresses a system actively controlling request flow, visually communicating throttling, boundary enforcement, and controlled access.

The design is protocol‑agnostic and vendor‑neutral, suitable for API documentation, backend workflows, developer dashboards, integration diagrams, and any environment where request limits or throttling states need to be shown.  
Its abstraction ensures clarity at small sizes and fits naturally into enterprise technical systems without implying any specific technology.

## Geometric Meaning  
The left‑side circle represents the incoming request stream — valid, structured, and reaching the system without authentication or transport issues.

The central boundary element expresses the enforcement layer where rate‑limit logic is applied.

The right‑side geometry represents the controlled processing zone, visually indicating that the system is intentionally restricting throughput rather than failing due to an error or resource issue.

The geometry communicates intentional throttling, not malfunction.

## Rate‑Limit Enforcement Indicator  
This icon signals that the system is deliberately slowing or blocking requests to maintain stability, prevent overload, or enforce usage policies.

It is not a client error in the traditional sense; instead, it reflects a protective mechanism triggered by excessive request frequency.

The icon’s structure highlights the boundary where enforcement occurs, making it clear that the system is functioning as designed.

## Narrative Fit  
Use this icon in API documentation, SDK guides, developer dashboards, integration diagrams, and backend workflow maps to indicate throttling, quota enforcement, or request‑rate control.

It fits naturally into technical ecosystems where rate limits are part of operational policy rather than system failure.

The abstraction avoids implying any specific vendor or technology, making it suitable for enterprise‑level diagrams and cross‑platform documentation.
