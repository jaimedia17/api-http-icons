# 404 — Not Found Icon  
Resource Lookup Failure Beyond Valid Request

## Overview  
The **404 — Not Found** icon represents a request that is structurally valid and processed by the server, but the targeted resource cannot be located.  
Unlike 400‑class structural errors, the request itself is interpretable — the failure occurs at the **resource resolution layer**.  
This icon expresses that the server has reached the lookup stage but cannot find a matching resource at the requested location.

## Geometric Meaning  
- **Left Circle — Request Entity**  
  The request is valid, received, and understood.  
  The server can parse the method, path, and parameters.

- **Middle Gap — Missing Resource Path**  
  This gap represents the absence of a resolvable target.  
  The request flows forward, but there is no resource to bind to.

- **Right Ghost Frame — Nonexistent Resource Layer**  
  The faint or incomplete structure indicates a resource that is expected in principle but not present in reality.  
  It communicates “nothing here” rather than “access denied” or “blocked.”

## Not Found Indicator  
- **Broken Continuity Across the Resource Layer**  
  The visual break emphasizes that the failure is not at authentication or authorization, but at **location and existence**.  
  The request reaches the resource tier, but the lookup returns empty.

## Narrative Fit  
The icon mirrors the technical sequence of a 404 response:

1. The request arrives and is structurally valid.  
2. The server routes the request toward the intended resource.  
3. The resource lookup is performed.  
4. No matching resource is found at the requested path or identifier.  
5. The server returns **404 Not Found** without implying permission or identity failure.

This abstraction aligns with the true meaning of 404 in API and HTTP workflows:  
**the request is valid, but the target resource does not exist at the requested location.**
