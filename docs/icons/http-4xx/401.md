# 401 — Unauthorized Icon
Credential Failure at Authentication Boundary

## Overview  
The **401 — Unauthorized** icon represents a request that reaches the authentication boundary but cannot proceed because the credentials are missing, invalid, or rejected.  
Unlike structural errors, the request itself is well‑formed — the failure occurs at the identity layer.  
This icon expresses that the server cannot verify the requester and therefore denies access before any resource lookup.

## Geometric Meaning  
- **Left Circle — Request Entity**  
  Represents a structurally valid incoming request.  
  The request exists, is received, and is interpretable.

- **First Dash — Authentication Boundary**  
  This is the credential checkpoint.  
  The request reaches this boundary but cannot pass through it.

- **Right Segment — Protected Resource Layer**  
  Remains untouched.  
  In a 401 scenario, the server does not evaluate or access the resource because authentication has not succeeded.

## Unauthorized Access Indicator  
- **“X” Inside the Request Circle**  
  Indicates credential failure.  
  The rejection is tied to the identity state of the requester, not the structure of the request or the availability of the resource.  
  This placement communicates that the request is known but not authorized to proceed.

## Narrative Fit  
The icon mirrors the real technical sequence of a 401 response:

1. The request arrives and is structurally valid.  
2. The server checks for authentication credentials.  
3. Credentials are missing, invalid, or rejected.  
4. The request is blocked at the authentication boundary.  
5. The resource layer is never evaluated.  
6. The server returns **401 Unauthorized**.

This abstraction aligns with the true meaning of 401 in API and HTTP workflows:  
**access cannot be granted because the requester has not provided valid authentication.**
