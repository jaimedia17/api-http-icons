# 503 — Service Unavailable Icon  
Temporary Server Unavailability or Maintenance State

## Overview  
This icon represents the 503 Service Unavailable condition using the same abstract, protocol‑neutral visual language used throughout the API/HTTP icon system.  
A 503 error occurs when the server is temporarily unable to handle the request.  
The request is valid, the server is reachable, and the server boundary is intact — but the server is offline, overloaded, or undergoing maintenance.

The icon expresses this by placing a striped offline pattern inside the server layer, indicating temporary unavailability rather than a permanent or internal failure.

## Geometric Meaning  
The circle on the left represents the incoming request.  
The request successfully reaches the server’s evaluation boundary, showing that the server is online and capable of receiving traffic.

Inside the server block, a diagonal striped pattern indicates that the server cannot process the request at this time.  
The resource layer on the right remains untouched, symbolizing that the server never reaches the stage of interacting with the target resource due to its temporary offline state.

## Temporary Unavailability Indicator  
The diagonal stripes inside the server block are the core semantic signal of the 503 condition.  
Unlike the fragmented shapes used in 500 or 502 errors, which represent internal or upstream failures, the striped pattern communicates a temporary pause or offline state.

This abstraction aligns with real 503 scenarios such as maintenance windows, overload protection, or scheduled downtime.  
The request path remains intact, reinforcing that the issue is not with the request or the gateway, but with the server’s current availability.

## Narrative Fit  
The icon mirrors the real technical sequence of a 503 error.  
The request is received by the server, but the server is temporarily unable to process it.  
No internal crash occurs, and no upstream dependency fails — the server simply cannot serve the request at this moment.

The resource layer remains irrelevant, and the server returns a Service Unavailable response.  
This abstraction aligns with how APIs and backend systems communicate temporary outages or maintenance states.
