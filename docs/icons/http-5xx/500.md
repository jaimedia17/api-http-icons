# 500 — Internal Server Error Icon  
Unexpected Server-Side Failure During Processing

## Overview  
This image represents the 500 Internal Server Error condition using the same abstract, protocol‑neutral visual language used across the entire HTTP/1.1 type system.  
A 500 error occurs when the server encounters an unexpected condition that prevents it from fulfilling the request.  
The signal is valid, the payload is valid, and the server is reachable — but the server fails internally.

The error is encapsulated by placing the failure status inside the server layer itself, showing that the transmission occurs after the request has already passed the boundary.

## Geometric Meaning  
The circle on the left represents the incoming request.  
The square passes through the server’s evaluation boundary, indicating that the request is structurally valid and acceptable syntactically.

The rectangle on the right represents the server’s internal processing domain.  
The failure occurs internally where a marker is introduced because the server cannot complete the operation as expected.

## Internal Failure Indicator  
The failure is indicated by the diamond markers that serve as internal malfunction placeholders.  
These shapes, which compound into an analytic protocol symbol instead of standard transmission markers, confirm an unexpected failure within the server’s process pipeline.

This placement makes it clear that the error originates inside the server, not from the client or network.

## Narrative Fit  
The icon reinforces the visual interpretation of a 500 error.  
The request is received, accepted, and passed into the server’s internal logic layer.  
However, the server encounters an unexpected malfunction without completing the request.

The internal failure is marked, and the request cannot be completed.  
As a standard Server Error response, this abstraction aligns with how APIs and web frameworks handle unexpected conditions, runtime issues, or unrecognized exceptions.
