# 502 — Bad Gateway Icon  
Upstream Server Failure Beyond the Gateway Layer

## Overview  
This icon represents the 502 Bad Gateway condition using the same abstract, protocol‑neutral visual language used throughout the API/HTTP icon system.  
A 502 error occurs when a server acting as a gateway or proxy receives an invalid or failed response from an upstream server.

The request is valid, the gateway is reachable, and the gateway attempts to forward the request — but the upstream system fails.  
The icon expresses this by placing the failure inside the upstream layer, not the gateway itself.

## Geometric Meaning  
The circle on the left represents the incoming request.  
The request successfully reaches the gateway layer, represented by the first bar.

The gateway then attempts to forward the request to the upstream server, represented by the second bar.  
The failure is placed inside the upstream server block, indicating that the gateway itself is functioning correctly but the upstream system returned an invalid, incomplete, or erroneous response.

The resource layer remains untouched because the upstream failure prevents the request from reaching its final destination.

## Upstream Failure Indicator  
The geometric fragments inside the upstream block represent a malfunction beyond the gateway’s control.  
Unlike a 500 error, where the server itself fails internally, the 502 error shows that the gateway is healthy but the next server in the chain is not.

This distinction is expressed by placing the failure in the second processing layer rather than the first.  
The request path is intact up to the gateway, reinforcing that the breakdown occurs further downstream.

## Narrative Fit  
The icon mirrors the real technical sequence of a 502 error.  
The request is received by the gateway, the gateway forwards it, and the upstream server returns an invalid or failed response.

The gateway cannot complete the operation and returns a Bad Gateway response.  
This abstraction aligns with how load balancers, proxies, and API gateways surface upstream outages or misconfigurations in real distributed systems.
