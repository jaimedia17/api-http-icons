# 504 — Gateway Timeout Icon  
Upstream Response Delay Beyond the Gateway Layer

## Overview  
This icon represents the 504 Gateway Timeout condition using the same abstract, protocol‑neutral visual language used throughout the API/HTTP icon system.  
A 504 error occurs when a server acting as a gateway or proxy does not receive a timely response from an upstream server.

The request is valid, the gateway is functioning, and the upstream server is reachable — but the upstream system takes too long to respond.  
The icon expresses this by placing a timeout pattern inside the upstream layer, showing that the delay occurs beyond the gateway.

## Geometric Meaning  
The circle on the left represents the incoming request.  
The request successfully reaches the gateway boundary, represented by the first bar.

The gateway forwards the request to the upstream server, represented by the second bar.  
Inside the upstream block, a timeout pattern indicates that the upstream system failed to respond within the required time window.

The resource layer on the right remains untouched, symbolizing that the request never reached the final processing stage due to the upstream delay.

## Upstream Timeout Indicator  
The timeout pattern inside the upstream block is the core semantic signal of the 504 condition.  
Unlike the fragmented shapes used in 500 or 502 errors, which represent internal or upstream failures, the timeout pattern communicates delay rather than malfunction.

The gateway is functioning correctly, and the request path is intact — the upstream server simply did not respond in time.  
This abstraction aligns with real 504 scenarios such as slow upstream services, overloaded dependencies, or network latency in distributed systems.

## Narrative Fit  
The icon mirrors the technical sequence of a 504 error.  
The request is received by the gateway, forwarded to the upstream server, and then stalls due to a timeout.

No internal crash occurs, and no invalid upstream response is returned — the upstream system simply takes too long.  
The gateway returns a Gateway Timeout response.

This abstraction aligns with how load balancers, proxies, and API gateways surface latency‑related failures in distributed architectures.
